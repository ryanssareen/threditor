# Threditor - Token Optimization Rules

**3D Minecraft Skin Editor: Next.js 15 + React Three Fiber + Zustand**

## Quick Start Commands

```bash
# Full status (RUN THIS FIRST every session)
tree -L 2 -I 'node_modules|.next' && \
echo "\n=== GIT ===" && git status --short && \
echo "\n=== DEPS ===" && npm list --depth=0 | head -15

# Dev server
npm run dev

# Type check
npx tsc --noEmit

# Tests
npm test -- --coverage --passWithNoTests

# Build
npm run build
```

## Project-Specific Shortcuts

### Find R3F/Three.js Usage
```bash
grep -rn "useFrame\|useThree\|Canvas" app/ lib/ --include="*.tsx" --include="*.ts"
grep -r "from 'three'" lib/ app/
```

### Check Zustand Store
```bash
grep -A 50 "create<" lib/editor/store.ts
```

### UV Mapping Constants
```bash
grep -B 2 -A 10 "CLASSIC_UVS\|SLIM_UVS" lib/three/geometry.ts
```

### Texture Pipeline
```bash
grep -rn "CanvasTexture\|needsUpdate\|texture.image" lib/editor/
```

### Component Locations
```bash
find app/ -name "*.tsx" -type f
grep -r "export.*function.*Props" app/ --include="*.tsx"
```

### M4 Specific (2D↔3D Paint Bridge)
```bash
# Pointer events
grep -rn "onPointer" app/ lib/

# Raycast usage
grep -rn "raycaster\|intersects" lib/

# Hover state
grep -rn "hovered\|hoveredPixel" lib/
```

## NEVER Waste Tokens On

❌ `view` to explore structure → use `tree` or `find`
❌ `view` package.json → use `npm list`
❌ `view` to find functions → use `grep -rn`
❌ `view` to check git → use `git status --short`
❌ Read whole file for one thing → use `grep` or `sed -n`

## Token Budget Guidelines

- **Quick lookup**: 1 shell command (50-100 tokens)
- **Code review**: 1-2 tool calls (800-1200 tokens)
- **Feature impl**: 5-10 tool calls (4000-8000 tokens)
- **Major refactor**: 10-20 tool calls (8000-15000 tokens)

**If using more → you're doing it inefficiently**

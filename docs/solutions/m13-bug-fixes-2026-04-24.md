## Problem 1: Gallery Cards Have No 3D Preview

**Current:** Static thumbnail image only  
**Expected:** Hoverable 3D model preview  

**Fix created:** `/app/gallery/_components/SkinPreview3D.tsx`

### Update SkinCard.tsx

Replace the thumbnail `<img>` with a dynamic 3D preview that shows on hover:

```tsx
// app/gallery/_components/SkinCard.tsx
'use client';

import dynamic from 'next/dynamic';
import { useState } from 'react';

// Lazy-load the 3D preview (three.js is ~600 KB)
const SkinPreview3D = dynamic(
  () => import('./SkinPreview3D').then(mod => ({ default: mod.SkinPreview3D })),
  { ssr: false }
);

export function SkinCard({ skin, ... }: Props) {
  const [show3D, setShow3D] = useState(false);

  return (
    <article
      onMouseEnter={() => setShow3D(true)}
      onMouseLeave={() => setShow3D(false)}
      ...
    >
      <Link href={`/skin/${skin.id}`}>
        <div className="relative flex items-center justify-center bg-ui-base p-4 h-48">
          {show3D ? (
            <SkinPreview3D
              skinUrl={skin.storageUrl}
              variant={skin.variant}
              className="absolute inset-0"
            />
          ) : (
            <img
              src={skin.thumbnailUrl}
              alt={skin.name}
              width={128}
              height={128}
              loading="lazy"
              style={{ imageRendering: 'pixelated' }}
              className="h-32 w-32"
            />
          )}
        </div>
      </Link>
      {/* rest unchanged */}
    </article>
  );
}
```

**Why lazy load:**
- three.js bundle: ~600 KB
- Gallery may show 20+ cards
- Only load when user hovers (progressive enhancement)

## Problem 3: Download Fails with "Bucket not found"

**Error:** 
```json
{
  "statusCode": "404",
  "error": "Bucket not found",
  "message": "Bucket not found"
}
```

**Root cause:** Supabase Storage bucket `skins` doesn't exist

### Fix: Create Supabase Bucket

**Step 1: Create Bucket**

1. Go to: https://supabase.com/dashboard
2. Select your project
3. Sidebar: **Storage**
4. Click: **New bucket**
5. Settings:
   - Name: `skins`
   - Public: **Yes** ✅
   - File size limit: `500 KB`
   - Allowed MIME types: `image/png,image/webp`

**Step 2: RLS Policies**

After creating the bucket, add these policies:

**Policy 1: Public Read**
```sql
CREATE POLICY "Anyone can read skins"
ON storage.objects FOR SELECT
USING (bucket_id = 'skins');
```

**Policy 2: Authenticated Upload (Own Folder)**
```sql
CREATE POLICY "Users can upload to own folder"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'skins'
  AND auth.uid()::text = (storage.foldername(name))[1]
);
```

**Policy 3: Authenticated Delete (Own Files)**
```sql
CREATE POLICY "Users can delete own files"
ON storage.objects FOR DELETE
USING (
  bucket_id = 'skins'
  AND auth.uid()::text = (storage.foldername(name))[1]
);
```

**Step 3: Verify**

Test the bucket exists:
```bash
curl https://[your-project-id].supabase.co/storage/v1/object/public/skins/
```

Expected: `[]` (empty array, not 404)

**Step 4: Environment Variables**

Verify `.env.local` has:
```env
NEXT_PUBLIC_SUPABASE_URL=https://[your-project-id].supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_BUCKET_NAME=skins
```

## Performance Notes

**Bundle size impact:**
- three.js: ~600 KB (lazy-loaded on hover)

- **Total:** ~750 KB (only loaded when user interacts)

- 20 cards visible → all show thumbnails (fast)
- User hovers 1 card → loads 3D preview for that card only
- Subsequent hovers → three.js already cached (instant)

**Skin detail optimization:**
- Always load 3D (user expects interactivity)
- Loading spinner while three.js downloads
